## Commands

### Development
```bash
# Start Metro bundler (Expo Go dev mode)
ANDROID_HOME=~/Library/Android/sdk npx expo start

# Start and auto-open on Android emulator
ANDROID_HOME=~/Library/Android/sdk npx expo start --android

# Clear Metro cache (fixes most bundling errors)
ANDROID_HOME=~/Library/Android/sdk npx expo start --clear
```

### Opening in Expo Go on emulator
After Metro is running on port 8081, the emulator uses `10.0.2.2` (not `localhost`) to reach the host:
```bash
adb shell am start -a android.intent.action.VIEW -d "exp://10.0.2.2:8081" host.exp.exponent
```

### Building release APK / AAB
Requires Java 17 (`/opt/homebrew/opt/openjdk@17`). Android Studio's bundled JDK (Java 21) does **not** work — Gradle requires Java 17.
```bash
export JAVA_HOME="/opt/homebrew/opt/openjdk@17"
export ANDROID_HOME=~/Library/Android/sdk
cd android
./gradlew assembleRelease   # → android/app/build/outputs/apk/release/app-release.apk
./gradlew bundleRelease     # → android/app/build/outputs/bundle/release/app-release.aab
```

### Install APK on emulator
```bash
adb install -r android/app/build/outputs/apk/release/app-release.apk
adb shell am start -n com.funnyapp.calculator/.MainActivity
```

### Regenerate native Android project
Run after changing `app.json` (icon, package name, permissions, etc.):
```bash
ANDROID_HOME=~/Library/Android/sdk npx expo prebuild --platform android --no-install
```
After prebuild, verify the release signing config is still present in `android/app/build.gradle` — prebuild can reset it.

## Architecture

**Expo SDK 51 / React Native 0.74** managed workflow, prebuilt to a native Android project.

### Data flow
`useCalculator` (reducer) → `App.tsx` (state + layout) → `CalcButton` (press handling) → `useSounds` + `FlyingEmoji`

- **`hooks/useCalculator.ts`** — pure reducer, all calculator logic. 9-digit input cap. Returns `displayValue`, `expression`, `activeOperator`, `isResult` plus action callbacks.
- **`hooks/useSounds.ts`** — generates WAV files programmatically at runtime (no audio assets). Writes them to `FileSystem.cacheDirectory` via `expo-file-system`, loads with `expo-av`. Each of the 10 digits has a unique base pitch on the C4–D5 scale; operators, equals, clear, and error each have a distinct synthesized sound.
- **`components/CalcButton.tsx`** — receives `soundEnabled` and `animEnabled` props; skips animations/emojis/sounds when disabled. Uses `measureInWindow` for emoji spawn position — guards against NaN/zero values with a fallback.
- **`components/FlyingEmoji.tsx`** — absolutely positioned, driven by `Animated`. `left`/`top` come from `CalcButton`'s `measureInWindow` callback.
- **`components/Display.tsx`** — receives `animEnabled`; skips all `Animated` sequences when false.

### Signing
Release keystore is at `funny-calculator.keystore` (excluded from git). Credentials: alias `funny-calculator`, storepass/keypass `funnycalc123`. Referenced in `android/app/build.gradle` under `signingConfigs.release`.

### Known quirks
- The `android/` directory is generated by `expo prebuild` and committed. Re-running prebuild resets `android/app/build.gradle` — re-apply the release `signingConfigs` block manually if that happens.
- `expo start --android` fails to auto-launch Expo Go if it isn't installed. Install Expo Go from the Play Store on the emulator, then use the `adb shell am start` command above.
- Emulator must use `10.0.2.2` to reach the Metro bundler on the host — `localhost` and the LAN IP do not work inside the emulator.
